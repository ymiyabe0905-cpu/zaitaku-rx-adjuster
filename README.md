# 処方日数アジャスター（試作版 v0.1）

処方箋の**日数調整・残薬調整・終了日合わせ・曜日固定薬・隔日薬・月固定薬**の確認を、すばやく正確に行うためのブラウザアプリです（在宅・外来どちらでも使えます）。

- フロントエンドのみで動作（**サーバー・ログイン不要**）
- 日付計算はローカルの JavaScript で実行（外部送信なし）
- React + TypeScript + Vite 構成
- 8ビット・レトロゲーム風の見た目（公式キャラ・ロゴ・ステージの再現はしていません）

> ⚠ **この計算結果は確認用です。最終判断は医師・薬剤師が行ってください。**（画面上部にも常時表示）

## 1. 起動方法

### 必要環境
- Node.js 18 以上（動作確認: Node.js 24）

### 手順

```bash
# このフォルダ（zaitaku-rx-adjuster）に移動して
npm install      # 初回のみ。依存パッケージを取得
npm run dev      # 開発サーバー起動
```

起動後、ターミナルに表示される URL（通常 `http://localhost:5173/`）をブラウザで開きます。

### 本番ビルド / プレビュー

```bash
npm run build    # 型チェック + 静的ファイルを dist/ に生成
npm run preview  # dist/ をローカルサーバーで確認
```

ビルド成果物は `dist/` に出力されます（`base: './'` 設定済みのため、`dist/index.html` をそのままブラウザで開いても概ね動作します。フォントはオンライン時のみ読み込み、オフラインでも読みやすい代替フォントにフォールバックします）。

## 3. 計算ルール（重要）

### 毎日服用薬は「服用回数ベース」で計算
日数ではなく服用回数を数え、最後に処方日数へ変換します。

```
必要錠数   = 必要服用回数 × 1回量
1日量(錠)  = 1日の服用回数 × 1回量
不足錠数   = max(0, 必要錠数 − 残薬)
処方日数   = ceil(不足錠数 ÷ 1日量)     ← 切り上げ。錠数と日数は別物
処方で出る錠数 = 処方日数 × 1日量
余る見込み = 残薬 + 処方で出る錠数 − 必要錠数
```

### 服用タイミング（用法）
用法は **朝・昼・夕・就寝前** の4時点の自由な組み合わせで表現します（食前/食後は区別しません。日数・回数計算に影響しないため）。
朝夕・毎食（朝昼夕）・朝昼・昼夕・朝就寝前・朝昼夕就寝前など、少数派のパターンも4つのチェックで表現でき、よく使う用法はプリセットボタンでワンタップ選択できます。
**開始タイミング**を指定でき、初日が部分回（例: 毎食を夕から開始 → 初日は1回分）になるケースを正しく数えます。

### 総合計算：定期薬の「飲み終わり日時」を回数ベースで算出
定期薬は **用法＋開始タイミング＋処方日数** から、実際に飲み切る最後の服用日・タイミングを**回数ベース**で計算します。
追加薬はその「飲み終わり日時」に合わせて必要回数・錠数・処方日数を求めます（追加薬の終了タイミングは定期薬から自動決定）。

```
定期薬の総服用回数 = 処方日数 × 1日の服用回数
定期薬の飲み終わり = 開始タイミングから総服用回数だけ進めた服用日・スロット
```

#### 計算例（総合計算タブのサンプル）
定期薬（朝昼夕・**6/17 夕開始・14日分**）の場合、暦日の終了日は6/30ですが、回数ベース（42回）で飲み切る飲み終わりは **7/1 昼** になります。
追加薬（朝昼夕・6/17 夕開始・1錠）をこれに合わせると：

| 項目 | 値 |
|---|---|
| 定期薬の飲み終わり | 7/1 昼 |
| 追加薬の必要服用回数 | 42回 |
| 追加薬の必要錠数 | 42錠 |
| 処方上必要な日数 | **14日分** |

このように暦日の終了日（6/30）と、回数ベースの飲み終わり（7/1昼）のズレを正しく反映します。必要錠数と処方日数は別の値として扱い、不足錠を1日量で割って切り上げます。

### 固定パターン（隔日・4週ごと・曜日固定・毎月○日）
日数ではなく、**期間内の実際の服用日を列挙して回数を数えます**。
```
必要錠数 = 服用回数 × 1回量
不足数   = max(0, 必要錠数 − 残薬)
```
- 隔日 = 1日おき（＝2日ごと、step 2日）。
- 4週ごと = 28日固定（step 28日）。
- 曜日固定：複数曜日を選択可（例: 火木、月水金）。
- 毎月○日：カレンダー上の同じ日（4週ごととは別物）。その月に無い日（例: 31日）はスキップ。

### インスリン残数
インスリン1剤ごとに、残量から「あと何日分」「持たせたい日まで持たせる追加本数」「必要な注入針本数」を計算します。

- **1回分として使えるかは、投与量ではなく「空打ち込みの実消費単位」で判定します。**
  - 空打ちを含める場合：`実消費単位 = 投与量 + 空打ち単位`、含めない場合：`実消費単位 = 投与量`。
  - 例：投与量12・空打ち2・使用中ペン残13単位 → 実消費14単位に満たないため**使用不可**。残14単位なら**使用可**。
- **端数の扱い**：残量は「使用中ペン → 未使用ペン」の順に消費します。あるペン（または使用中ペン）の残量が次回注射の実消費単位に満たない場合、その端数は**使用不可端数**とし、次のペンを開ける前提で計算します。
- **各回単位入力モード**：朝・昼・夕・寝る前の単位数を個別に判定します。0単位のタイミングは注射しない扱い。各回ごとに空打ちを足した実消費単位で、注射予定を順番に並べて残量から消費します（単純な1日量割りはしません）。
- **追加インスリン本数**：単純な不足単位 ÷ 1本単位ではなく、実際の注射予定を順番に満たせる最小本数で計算します（1本末尾の端数が次の1回分に満たないと使えないため）。
- **注入針**：注射1回につき1本（空打ちの有無や再使用は考慮しません）。`必要注入針 = 必要注射回数`。
- 現在の残量で使える量は「○日分」だけでなく必ず「○日分＋○回分」で表示します。
- **スライディングスケールや血糖値に応じた可変単位の指示には、今回の自動計算は対応していません。**
- **使用開始後期限による廃棄判定（簡易）**：「使用開始後期限を考慮する」をオンにすると（初期値オン・28日/56日/その他から選択）、**1本を使い切るまでの日数が期限を超える分を「廃棄見込み」として使用不可**に扱います。
  - `1日消費単位 = 1日の注射回数 × 1回あたり実消費単位`（各回入力モードは1日分の実消費単位の合計）。
  - `期限内使用可能単位数 = 使用開始後使用可能日数 × 1日消費単位`、`1本あたり実使用可能単位数 = min(1本あたり単位数, 期限内使用可能単位数)`。
  - 例：1本300単位・1回4＋空打ち2＝6単位・1日1回・期限28日 → 28日で168単位使用可能、残り132単位は廃棄見込み（必要本数の計算に含めない）。
  - **1本ごとの使用開始日は管理しません**（使用中ペンの開始日入力も不要）。1本を使い切るまでに期限を超える場合に超過分を廃棄見込みとして扱う簡易計算です。
  - 使用開始後の使用可能期間は製剤ごとに異なるため、添付文書・薬剤情報・施設の管理方法を必ず確認してください。

### 吸入薬残数
確認できる残吸入数（残量カウンター）をもとに計算します。2モードを切り替えます。

- **定期吸入モード**：残吸入数と1日使用吸入数から、持たせたい日までの不足・追加キット数を計算します。
  - `1日使用吸入数 = 1回あたり吸入数 × 1日の使用回数`、`現在の総使用可能吸入数 = 未使用キット数 × 1キット総吸入数 + 使用中キットの残吸入数`
  - 必要日数は開始日を含めて計算（持たせたい日当日分まで）。
  - `不足吸入数 = 必要吸入数 − 総使用可能吸入数`、`追加キット数 = ceil(不足 ÷ 1キット総吸入数)`。
  - 現在の残量は「○日分＋○回分」で表示。**1回量に満たない端数吸入数は、1回分として不完全な端数として表示します**（例: 1回2吸入・1日2回・残7吸入 → 1日分＋1回分＋端数1吸入）。
- **頓用吸入モード**：**前回〜今回の使用量をもとにした参考計算**です（厳密な日数保証ではありません）。
  - `使用吸入数 = 前回残 + 追加キット × 1キット総吸入数 − 今回残`（途中追加に対応）、`1日平均 = 使用吸入数 ÷ 期間日数`。
  - 持たせたい日までは「翌日から持たせたい日まで」で見込み、平均使用量または手入力の1日見込みで必要吸入数を算出。画面に「頓用薬のため参考計算」と表示します。
- **カウンターなし吸入薬の理論残数計算には対応しません。**
- **空噴霧・試し噴霧・初回準備吸入の差し引き計算には対応しません。**

### 点眼薬残数
**確認用の概算**機能です。容量から滴数を自動換算して計算します。

- **滴数の換算**：`5mL＝100滴`、`2.5mL＝50滴`、その他は `floor(容量mL × 20)`。**1mLあたり20滴**として概算します（1本あたり滴数の自由入力は不可）。
- 使用中ボトルの残量は「滴数で入力」または「残量割合（ほぼ満量/3·4/1·2/1·4/少量/空）で入力」。**割合入力は概算**です（残量カウンターのような正確な値ではありません）。割合の滴数は切り捨て。
- **両眼点眼では1回使用滴数が2倍**になります（`1回使用滴数 = 対象眼数 × 1眼あたりの1回滴数`、両眼=2眼）。
- 必要日数は**開始日と持たせたい日をどちらも含めて**計算（`必要日数 = 持たせたい日 − 開始日 + 1`）。
- `不足滴数 = 必要滴数 − 総使用可能滴数`、`追加本数 = ceil(不足 ÷ 1本あたり滴数)`。
- 現在の残量は「○日分＋○回分」で表示。**1回量に満たない端数滴数は、1回分として不完全な端数として表示します**（例: 両眼1滴×1日2回・残10滴 → 2日分＋1回分）。
- **開封後期限・点眼失敗回数は計算しません。** **予備日数・予備滴数の計算も行いません。**
- サンプルケースは画面に表示しません（テストコードでのみ近い値を使用）。

## 5. 注意事項・前提

- 端数のある1回量（例: 0.5錠）も入力可能です。
- 「毎月○日」でその月に存在しない日（例: 31日で30日までの月）は**スキップ**します。
- サマータイムの無い日本のローカル日付を前提に日数計算しています。
- あくまで確認用の試作版です。診療報酬・算定要件の判定は行いません。
